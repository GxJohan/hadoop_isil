REGLA IMPORTANTE

Si el comando inicia con docker, normalmente se ejecuta desde tu computadora.
Si el comando inicia con hdfs dfs, normalmente se ejecuta dentro del contenedor namenode.

A. DESDE TU COMPUTADORA - VERIFICAR DOCKER

docker --version
docker compose version
docker ps

B. DESDE TU COMPUTADORA - DESCARGAR REPOSITORIO

git clone https://github.com/big-data-europe/docker-hadoop.git
cd docker-hadoop

C. DESDE TU COMPUTADORA - LEVANTAR HADOOP
Estando dentro de hadoop_isil/docker-hadoop:

docker compose up -d
docker ps

D. DESDE TU COMPUTADORA - ENTRAR AL CONTENEDOR NAMENODE

docker exec -it namenode bash

E. DENTRO DEL CONTENEDOR NAMENODE - PRIMERA PRUEBA HDFS

echo "hadoop big data hadoop universidad datos datos hdfs" > /tmp/texto.txt
hdfs dfs -mkdir -p /input
hdfs dfs -put -f /tmp/texto.txt /input/
hdfs dfs -ls /input
hdfs dfs -cat /input/texto.txt
exit

F. DESDE TU COMPUTADORA - VERIFICAR NOMBRE DEL CSV PESADO

Archivo descargado desde:
https://examplefile.com/code/csv/200-mb-csv

Mac / Linux:

ls -lh

Windows PowerShell:

dir

Si tiene otro nombre, renombrar:

Mac / Linux:

mv nombre_original.csv datos_200mb.csv

Windows PowerShell:

ren nombre_original.csv datos_200mb.csv

G. DESDE TU COMPUTADORA - COPIAR CSV PESADO AL CONTENEDOR

Si estas en hadoop_isil:

docker cp datos_200mb.csv namenode:/tmp/datos_200mb.csv

Si estas dentro de docker-hadoop:

docker cp ../datos_200mb.csv namenode:/tmp/datos_200mb.csv

Luego entrar:

docker exec -it namenode bash

H. DENTRO DEL CONTENEDOR NAMENODE - SUBIR CSV PESADO A HDFS

hdfs dfs -mkdir -p /data
hdfs dfs -put -f /tmp/datos_200mb.csv /data/
hdfs dfs -ls -h /data
hdfs dfs -head /data/datos_200mb.csv
exit

I. VERIFICACION EN EL NAVEGADOR

Abrir:

http://localhost:9870

Buscar:

Utilities -> Browse the file system

Revisar:

/input
/data

Tambien puedes abrir:

http://localhost:8088

J. DESDE TU COMPUTADORA - APAGAR HADOOP

Estando dentro de hadoop_isil/docker-hadoop:

docker compose down
